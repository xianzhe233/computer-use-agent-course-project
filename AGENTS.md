# 项目协作指引

本文件是本仓库的入口地图，用来告诉后续 AI / 协作者：先看什么、规则放哪、做完后要更新什么。

## 必读顺序

1. 先读 `AGENTS.md`
2. 再读 `doc/architecture.md`
3. 再读 `doc/project/project-design.md`
4. 如果任务涉及外部方案借鉴、工具设计参考或 benchmark 调研，再看 `doc/references/README.md`
5. 开工前读 `doc/task-workflow.md`
6. 再看 `doc/progress.md`
7. 如果任务涉及 `git / commit / push`，再读 `doc/commit-conventions.md`
8. 接任务时复制 `doc/work-template.md`
9. 规则有变化时追加 `doc/log.md`

## 备注

`AGENTS.md` 仅作为项目地图，不存放详细信息，应该始终小于 100 行。
