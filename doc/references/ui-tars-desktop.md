# UI-TARS Desktop 参考笔记

## 来源

- GitHub：`https://github.com/bytedance/ui-tars-desktop`

## 大致结构

UI-TARS Desktop 更像一个完整的 **桌面 GUI agent 产品形态**，而不是单篇方法文档。它通常值得关注的结构包括：

- 桌面应用入口
- 本地/远程电脑连接或运行环境
- GUI agent 执行流程
- 交互界面与任务展示
- 历史记录、配置、运行管理

它提供的是“用户实际如何使用桌面 agent”的产品视角。

## 值得借鉴的部分

1. **桌面 agent 的产品组织方式**
   - 适合参考 demo 应该怎么展示，而不只是底层工具怎么写。
2. **任务执行界面与交互流程**
   - 对我们后期做展示 demo、整理 run 记录很有帮助。
3. **本地桌面环境中的 agent 使用体验**
   - 可帮助我们思考如何包装成更完整的展示系统。
4. **运行状态和历史记录呈现**
   - 与我们的截图、轨迹保存机制可以形成呼应。

## 对我们项目的帮助

- **MVP 5** 的重点参考对象。
- 对“如何把功能做成可展示 demo”特别有帮助。
- 可以帮助我们思考：
  - 任务开始界面怎么组织
  - 运行过程怎么展示
  - 历史截图和轨迹怎么呈现给验收者

## 可优先改写到我们项目里的点

- demo 的展示形态
- run 历史和操作记录的呈现方式
- 用户输入任务到开始执行的交互流程

## 参考时的注意事项

- UI-TARS Desktop 更偏产品化和应用形态，不能直接替代我们的核心系统设计。
- 我们应优先吸收其展示层和交互层思路，而不是盲目追求功能规模。
