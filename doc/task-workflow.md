# 任务工作流

每次接到任务时，默认按以下流程执行；除非用户明确要求跳过，否则不要省略。

## 步骤

1. **创建任务笔记**
   - 从 `doc/work-template.md` 复制一份到 `doc/current_tasks/`
   - 命名格式：`YYYY-MM-DD-HHMM-任务.md`
   - 先填写元信息，状态设为“进行中”

2. **理解上下文**
   - 先读 `AGENTS.md`
   - 再读 `doc/architecture.md` 与 `doc/progress.md`
   - 如果任务会改规则或目录，提前记下需要同步更新的文档

3. **拆解任务**
   - 把任务拆成可执行、可勾选的步骤
   - 使用 `- [ ]` 格式列出
   - 有假设、风险、阻塞时直接写进任务笔记

4. **执行任务**
   - 按清单逐项完成，完成一项勾选一项
   - 每完成一个子任务，立刻更新任务笔记中的勾选状态和对应“执行记录”，不得在任务结束后集中补写
   - 如果任务拆分在执行中发生变化，先更新任务清单，再继续执行
   - 关键操作和验证结果写入“执行记录”
   - 若任务卡住，将状态改为“阻塞”，并保留在 `doc/current_tasks/`

5. **收尾归档**
   - 结构变化：更新 `doc/architecture.md`
   - 进度变化：更新 `doc/progress.md`
   - 规则变化：更新 `doc/log.md`
   - 任务完成后将状态改为“已完成”，并移入 `doc/agent_log/`

## 最低要求

- 不跳过任务笔记
- 任务笔记必须伴随执行实时更新，而不应该最后统一补写
- 不事后伪造执行记录
- 不把已完成任务留在 `doc/current_tasks/`
- `doc/log.md` 只记规则/模板变化，不记普通开发进度
